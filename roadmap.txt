PHASE 1 — THEORETICAL FOUNDATION  (Months 1-3)
================================================================================

Most important phase. Without proofs you have a workshop paper.

------------------------------------------------------------
1.1  Formalize the Problem (Week 1-2)
------------------------------------------------------------
Write the formal problem statement before writing any more code.

  Define:
    S = {(x_t, y_t, a_t)}        data stream, a_t is protected attribute
    theta_t                        model parameters at batch t
    F(theta_t; W_t) <= tau         fairness constraint on sliding window W_t
    U subset W_t                   harmful points to remove
    L_task(theta_new) <= L_task(theta_old) + eps_acc   utility constraint

  This framing lets you state the main theorem before proving it.

------------------------------------------------------------
1.2  Certified Unlearning Bound (Week 2-5)
------------------------------------------------------------
Your strongest theoretical contribution.

  Goal: Prove Newton-step unlearning achieves (eps, delta)-certified
  unlearning in the streaming convex setting.

  Key result to prove:
    ||theta_unlearn - theta_retrain||_2 <= O(|U| / (n * lambda_min(H)))

  where:
    |U|            = number of unlearned points
    n              = effective training set size (replay buffer)
    lambda_min(H)  = smallest eigenvalue of Hessian (condition number)

  Why tractable:
    For logistic regression H is the empirical Fisher matrix.
    lambda_min(H) can be bounded if data is sub-Gaussian and replay
    buffer is drawn i.i.d. from the clean distribution.

  Papers to read first (in order):
    1. Guo et al. 2020 — "Certified Data Removal from Machine Learning Models"
    2. Sekhari et al. 2021 — "Remember What You Want to Forget"
    3. Neel et al. 2021 — "Descent-to-Delete: Gradient-Based Methods for
       Machine Unlearning"

  Note: your Newton-step already does this approximately. The theoretical
  work is formalizing what "approximately" means and bounding the error.

------------------------------------------------------------
1.3  Fairness Recovery Bound (Week 4-7)
------------------------------------------------------------
After unlearning U, prove how much fairness improves.

  Key result to prove:
    E[SPD(theta_new)] <= E[SPD(theta_old)] - Delta_F
                        + O(|U*|^(1/2) / n^(1/2))

  where U* is the set identified by influence scoring.

  The O(sqrt(|U*|/n)) term is LiSSA approximation error.

  Requires:
    - Bounding LiSSA approximation error (Koh & Liang 2017, Appendix)
    - Connecting influence scores to SPD gradient
    - Sub-Gaussian assumption on features

------------------------------------------------------------
1.4  CUSUM Detection Delay Bound (Week 6-8)
------------------------------------------------------------
Easiest of the four results. CUSUM theory is mature.

  From classical CUSUM theory (Lorden 1971):
    E[detection delay] = O(log(1/alpha) / Delta^2)

  where alpha = false alarm rate, Delta = SPD shift magnitude.

  Translate to your setting:
    - Define Delta as SPD change per batch after bias injection
    - Show your window_size and threshold tau achieve a specific
      (false alarm rate, detection delay) operating point
    - This is a ~1 page theoretical result

------------------------------------------------------------
1.5  Accuracy-Fairness-Unlearning Tradeoff (Week 8-10)
------------------------------------------------------------
Prove you cannot simultaneously have all three for free:
  (a) eps-certified unlearning
  (b) |SPD| <= tau fairness guarantee
  (c) accuracy loss <= delta

  This is an impossibility result. Prove a lower bound showing any
  algorithm satisfying (a) and (b) must incur at least Omega(f(eps,tau))
  accuracy loss. Then show your method achieves this bound tightly.

  Why this matters: impossibility results are highly valued at NeurIPS/ICML.
  They tell reviewers what is fundamentally achievable, not just what you did.

------------------------------------------------------------
2.1  TracIn for Approximate Influence (Week 6-10)
------------------------------------------------------------
TracIn (Pruthi et al., NeurIPS 2020) works for any differentiable model:

  TracIn(z, z_test) = sum_t  eta_t * grad_L(z, theta_t) . grad_L(z_test, theta_t)

  where theta_t are saved checkpoints during training.

  Implementation plan:
    1. Modify attribution/influence.py to support tracin backend
       alongside existing LiSSA backend
    2. Add FrameworkConfig flag: influence_method = "lissa" | "tracin"
    3. Save model checkpoints during streaming loop (every K batches)
    4. TracIn score = dot product of gradient traces over checkpoints

  This is ~200 lines of code and gives full neural net support.

------------------------------------------------------------
2.2  MLP Base Model (Week 8-12)
------------------------------------------------------------
Replace OnlineLogisticRegression with a small MLP:
  Architecture: input -> 128 -> ReLU -> 64 -> ReLU -> 1 -> sigmoid
  Training: mini-batch SGD, same interface as current model

  Key changes in models/base_model.py:
    - Add MLPModel class with same .predict(), .update(), .gradient() interface
    - gradient() via manual backprop or autograd
    - hessian_vector_product() approximated via finite differences
      OR replaced entirely by TracIn

  Why MLP not LightGBM:
    MLP is differentiable -> influence functions work
    LightGBM is not differentiable -> needs completely different approach
    Keep complexity manageable for first top-tier submission

3.2  Experimental Protocol (non-negotiable for top-tier)
------------------------------------------------------------

  Seeds:
    Run every experiment with 5 different random seeds.
    Report mean +/- standard deviation. Never report a single run.

  Statistical significance:
    Use Wilcoxon signed-rank test (non-parametric, paired) to compare methods.
    Report p-values. Claim significance only at p < 0.05.
    Report Cohen's d effect size alongside p-value.

  Streaming protocol (apply consistently across all datasets):
    Batches 1 to T*0.4       : clean warmup
    Batches T*0.4 to T*0.75  : bias injection window
    Batches T*0.75 to T      : clean recovery period

    This tests three things: detection, correction, AND recovery.
    Reviewers want to see SPD go up, get corrected, then come back down.

  Metrics to report in every table:
    AUC             primary performance metric
    SPD             fairness (statistical parity difference)
    EOD             fairness (equalized odds difference)
    Violations      number of batches exceeding threshold tau
    Unlearn events  number of triggered unlearning actions
    Cost (ms)       wall-clock time per unlearning event
    vs Retrain      ratio of unlearning cost to full retrain cost

This is actually a really strong research angle — and it's timely. What you're describing has a formal name: performative prediction combined with feedback loop bias amplification. It could significantly differentiate your paper.                  
                               
  ---                                                                                                                                                                                                                                                    
  The mechanism you're describing             
                                                                                                                                                                                                                                                         
  In deployed decision systems, the model's predictions become the training data for the next iteration:                                                                                                                                                 
                                                                                                                                                                                                                                                         
  Model predicts group A is high risk                                                                                                                                                                                                                    
      -> Group A gets denied loans / flagged                                                                                                                                                                                                             
          -> Group A has worse observed outcomes
              -> Next model sees group A as even higher risk
                  -> Bias compounds each iteration

  This is fundamentally different from your current bias injection (manual label flipping). The bias is endogenous — generated by the model itself. It's been shown to grow monotonically without intervention.

  ---
  Why this is novel for AFU

  Current AFU assumes bias arrives from outside (exogenous). No existing fairness unlearning paper formally models self-reinforcing bias from the model's own predictions. That gap is your contribution.

  The theoretical claim becomes much stronger:

  ▎ "Under performative feedback, SPD grows as O(t · α) without intervention, where α is the model's influence on future labels. We prove AFU bounds this growth to O(log t) by periodically breaking the feedback loop via selective unlearning."

  ---
  How to implement it

  Add a FeedbackLoopStreamLoader that wraps any existing loader:

  class FeedbackLoopStreamLoader:
      """
      Simulates a deployed model whose predictions influence future training labels.
   
      Mechanism:
          At each batch, a fraction `influence_rate` of future labels are
          replaced by the model's current predictions rather than ground truth.
   
          This mimics real deployment: if the model flags someone as high risk,
          they may not receive the loan/service, so their "true" outcome is
          never observed — the model's prediction IS the outcome.
   
      Bias amplification:
          If the model is already biased (higher P(y=1|A=0) than P(y=1|A=1)),
          the feedback loop makes group 0 look increasingly risky over time.
          SPD grows monotonically without external intervention.
      """

      def __init__(self, base_loader, model_ref, influence_rate=0.3, seed=42):
          self._base = base_loader
          self._model = model_ref      # live reference — predictions change as model updates
          self._rate = influence_rate  # fraction of labels replaced by model predictions
          self._rng = np.random.RandomState(seed)

      def stream(self, batch_size=500):
          for batch in self._base.stream(batch_size=batch_size):
              y_corrupted = batch.y.copy()

              # For `influence_rate` fraction of samples, replace ground truth
              # with model's current prediction (simulating "decision = outcome")
              mask = self._rng.rand(len(batch.y)) < self._rate
              if mask.any():
                  model_preds = (self._model.predict(batch.X[mask]) >= 0.5).astype(int)
                  y_corrupted[mask] = model_preds

              yield DataBatch(
                  X=batch.X, y=y_corrupted,
                  protected=batch.protected,
                  timestamp=batch.timestamp,
                  indices=batch.indices,
                  metadata={"feedback_corrupted": mask.sum(), "influence_rate": self._rate}
              )

  ---
  The experiment this enables

  Compare three conditions in your streaming setup:

  ┌───────────────────────┬─────────────────────────────────────────────────┐
  │       Condition       │          What happens to SPD over time          │
  ├───────────────────────┼─────────────────────────────────────────────────┤
  │ No feedback, no AFU   │ SPD stays flat (baseline)                       │
  ├───────────────────────┼─────────────────────────────────────────────────┤
  │ Feedback loop, no AFU │ SPD grows monotonically — proves the problem    │
  ├───────────────────────┼─────────────────────────────────────────────────┤
  │ Feedback loop + AFU   │ SPD growth bounded — proves AFU breaks the loop │
  └───────────────────────┴─────────────────────────────────────────────────┘

  This is a much more compelling empirical story than manual bias injection, because the bias emerges naturally from the system rather than being artificially inserted.

  ---
  Connection to the LLM model collapse literature

  Shumailov et al. 2023 ("The Curse of Recursion") showed that LLMs trained recursively on AI-generated data forget the tails of distributions — minority populations progressively disappear from what the model can represent. Your work is the
  decision-system analogue:

  - LLMs: minority tokens become underrepresented in generations
  - Decision systems: minority groups become increasingly misclassified

  This connection makes your paper relevant to a much wider audience than just fairness researchers.

  ---
  New theorem it enables

  Theorem (Feedback Loop Divergence without Intervention):
  Let α be the influence rate and SPD₀ be the initial disparity. Under the feedback loop model:

  E[SPD_t] >= SPD_0 * (1 + alpha)^t   [grows exponentially in t]

  Corollary (AFU breaks the loop):
  With AFU triggering every K batches when SPD > τ:

  E[SPD_t] <= tau + O(K * alpha)   [bounded, independent of t]

  This is a clean, provable result that directly motivates why continuous monitoring and unlearning is necessary — not just nice to have.
